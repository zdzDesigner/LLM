# Code Example Searcher (最佳实践搜索器)

## 📋 功能概述
这是一个**饱和式搜索专家**。当你对某个功能的实现方式感到模糊，或者想了解"最佳实践"时，它会同时启动多个智能体，分别深挖**本地代码库的历史模式**和**Github/文档的行业标准**，最后生成一份对比分析报告。

## 🎯 使用场景
- **技术选型前**：想看看业界怎么做，同时确认本地是否有类似轮子。
- **重构代码时**：寻找"正确"的写法，识别本地的"反模式"。
- **接手新项目**：快速了解项目里某种业务逻辑（如鉴权、日志）的惯用写法。
- **解决疑难杂症**：通过对比本地实现和官方Demo，找出逻辑漏洞。

## 📁 输入/输出

### 输入
- **模糊需求**：一段自然语言描述（中英文皆可）。

### 输出
- **Markdown 报告**：包含核心结论、本地现状、行业标准对比、综合建议。

## 🔧 核心机制
本 Skill 采用 **Map-Reduce** 模式：
1. **Map (并行)**: 
   - `librarian` (外部知识)
   - `explore` (内部知识)
   - `ast_grep` (结构化匹配)
2. **Reduce (聚合)**:
   - 对比差异
   - 识别差距
   - 生成建议
